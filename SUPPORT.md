# Support

Engramia is source-available software under [Business Source License 1.1](LICENSE.txt).

## How to get help

| Need | Where |
|------|-------|
| Bug report | [Open an Issue](https://github.com/engramia/engramia/issues/new?template=bug_report.yml) |
| Feature request | [Open an Issue](https://github.com/engramia/engramia/issues/new?template=feature_request.yml) |
| Security vulnerability | [Private advisory](https://github.com/engramia/engramia/security/advisories/new) or [support@engramia.dev](mailto:support@engramia.dev) |
| Documentation | [engramia.readthedocs.io](https://engramia.readthedocs.io) |
| Community questions | [GitHub Discussions](https://github.com/engramia/engramia/discussions) |
| Commercial licensing | [support@engramia.dev](mailto:support@engramia.dev) |
| Enterprise support / SLA | [support@engramia.dev](mailto:support@engramia.dev) |

## What is *not* supported

- External code contributions — see [CONTRIBUTING.md](CONTRIBUTING.md).
- Support for versions older than the current minor release — see [SECURITY.md](SECURITY.md).
- Custom integration work without a commercial engagement — see commercial licensing above.

## Response times

Bug reports and feature requests on GitHub Issues are reviewed on a best-effort
basis. Security vulnerabilities have defined SLAs — see [SECURITY.md](SECURITY.md).

Commercial support SLAs are negotiated per engagement.
